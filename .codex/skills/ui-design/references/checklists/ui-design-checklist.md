# UI Design Checklist

## Knowledge Check

- [ ] 表示項目、主要 action、button enablement を確認した
- [ ] state、variant、responsive、overflow risk を実装後確認観点として確認した
- [ ] `ui-design.md` は UI 要件契約と確認観点に限定した

## Common Pitfalls

- [ ] 実装前の見た目 artifact を必須 artifact として復活させなかった
- [ ] product frontend implementation に踏み込まなかった
- [ ] human が実装後に確認すべき visual polish を隠さなかった
