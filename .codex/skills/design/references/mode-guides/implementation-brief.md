# Design: implementation-brief

## Goal

- 実装 task の順序、ownership、validation を固定する

## Capture

- accepted scope
- task split
- validation commands
- diagram need
- architecture 変更時だけ `docs/architecture.md` と対象 D2 を `source_diagram_targets` に載せる
- review handoff condition

## Avoid

- broad refactor の誘発
- human 未承認の仕様追加
- architecture 成果物がないのに `source_diagram_targets` を埋めること
