# Requirements Design Checklist

## Knowledge Check

- [ ] capability、constraints、non-goals を分けた
- [ ] confirmed、inferred、open question を分けた
- [ ] scenario-design へ渡せる acceptance basis を作った

## Common Pitfalls

- [ ] product truth を発明しなかった
- [ ] implementation scope や file ownership を書かなかった
- [ ] 既存正本との衝突を隠さなかった
