# Skill Modification

## 目的

`skill-modification` は、`.codex` 配下の 作業流れ、skill、エージェント実行定義、ツール権限 を、実行時に再解釈が起きない契約として更新するための作業プロトコルである。

この skill は、変更内容を正しい規約へ配置し、責務境界、正本配置、入力、判断、出力、完了、停止の意味を揃えるために使う。
変更後の `.codex` は、別の Codex または agent が同じ境界、同じ入力、同じ判断、同じ停止条件で読める状態に戻す。

## 対応ロール

- この skill は人間から直接依頼された Codex 本体だけが使う。
- サブエージェントまたは別 agent から呼び出された場合は拒否する。
- この skill は `.codex` 配下の 作業流れ / skill / agent 変更作業を対象にする。

## 入力規約

- 入力は、変更目的、変更対象パス、非対象、残す正本、削除してよい対象を含む。
- skill / agent / ツール権限 を変更する場合は、変更対象名とファイルパスを必須にする。
- 削除がある場合は、削除理由と削除後の正本確認先を必須にする。
- 入力が不足する場合は、推測で変更せず、人間へ不足項目を返す。

## 外部参照規約

- skill 形式の正本は `.codex/skill-template.md` とする。
- エージェント実行定義の正本形式は `.codex/agent-template.toml` とする。
- agent 紐づけ と ツール権限 は エージェント実行定義 に置く。
- プロダクト 仕様の正本は `docs/` であり、この skill は プロダクト 仕様を変更しない。
- 外部正本が衝突する場合は停止し、衝突内容を人間へ返す。

## 内部参照規約

規約配置は次の表に従う。
この表は、変更内容をどの規約へ置くかを拘束する。

| 規約 | 書く内容 | 書かない内容 |
| --- | --- | --- |
| 入力規約 | 呼び出し元から受け取る入力、必須項目、承認状態、不足時の扱い | 判断基準、出力形、完了条件、停止条件 |
| 外部参照規約 | skill 外の正本、agent 実行定義、参照優先順位、外部正本衝突時の扱い | skill 内だけで完結する表、判断手順、出力項目 |
| 内部参照規約 | skill 内で使う表、DAG、分類表、各参照が拘束する判断 | 外部正本、手順、出力義務 |
| 判断規約 | 担当ロールが判断する基準、判断してよい範囲、判断してはいけない範囲 | 入力必須項目、返す出力の形、停止時の返却先 |
| 非対象規約 | この skill が扱わない成果物、判断、変更対象、他へ渡す対象 | 不足、衝突、境界違反などの停止条件 |
| 出力規約 | 返す情報、更新する成果物、置き場所、粒度、含めてはいけないもの | 判断基準、完了条件、停止条件 |
| 完了規約 | 完了とみなす状態、必要な根拠、検証、完了時に返す情報 | 非対象、停止条件、作業順序 |
| 停止規約 | 進めてはいけない状態、停止時に返す情報、戻し先、人間判断条件 | 扱わない対象だけの列挙、通常の判断基準 |

## 判断規約

- skill は作業プロトコルであり、手順、標準 型、参照タイミング一覧、知識範囲一覧を持たない。
- `.codex/skill-template.md` の `Maintenance（テンプレート外）` は、個別 skill や agent に書き込む内容ではない。
- agent は実行主体であり、エージェント実行定義とツール権限を持つ。
- 入力規約、非対象規約、出力規約、完了規約、停止規約は skill 側へ統合する。
- 非対象規約は、skill が扱わない成果物、判断、変更対象だけを書く。
- 停止規約は、不足、衝突、境界違反、戻し先、人間判断条件だけを書く。
- 変更内容を追加する規約は、内部参照規約の規約配置表で決める。
- 規約配置が曖昧な場合は、入力、判断、非対象、出力、完了、停止のどれを拘束する内容かで分類する。
- ファイル名、agent 名、skill 名、command 以外の英語由来名詞は、解釈の幅が狭い日本語へ置き換える。
- 構文項目は構文として扱い、本文の説明では書き込み許可、実行許可のような日本語名で書く。
- 英語名を併記する必要がある場合は、先に日本語の意味を書き、括弧内に残してよい分類を置く。
- 置換前に、解釈の幅がある語、残す固定名、置換後の日本語を一覧化する。
- 責務、ツール権限、入力、出力が分かれる場合は、分岐記述ではなく skill または agent の分割を選ぶ。
- 現行 作業流れ にない 成果物、廃止ファイル、旧参照、空実体、禁止文言だけの説明ファイルは残さない。
- 論理名と変更対象名は、同じ判断単位で対応が分かるように書く。
- 確認用ファイルは個別ファイルとして残さず、確認条件は skill の `完了規約` または `停止規約` へ統合する。
- agent 個別 JSON 規約は個別 JSON として残さず、入力、出力、完了、停止の規約は対応 skill へ統合する。
- 旧権限ファイルは残さず、実行権限と 書き込み範囲 は エージェント実行定義の 書き込み許可 / 実行許可 に置く。
- 作業流れ 本体 README は個別 skill の `外部参照規約` に入れない。

## 非対象規約

- プロダクトコード、プロダクトテスト、プロダクト仕様文書は変更しない。
- agent TOML の権限不足や所有者不明を推測で補わない。
- 削除理由または削除後の正本確認先がない削除は扱わない。

## 出力規約

- 出力は、更新する `.codex` 配下ファイル、削除するファイル、削除理由、検証方法を含む。
- 曖昧語を置換する場合は、検出した語、残す固定名、置換後の日本語を先に示す。
- skill 本体を更新する場合は、`.codex/skill-template.md` の分類順に合わせる。
- 非対象規約を追加または更新する場合は、停止規約へ混ぜた非対象を分けて示す。
- agent を更新する場合は、エージェント実行定義、紐づけ、ツール権限 の所有者を分けて示す。
- 削除対象がある場合は、削除後にどの正本を見ればよいかを示す。
- 出力に プロダクトコード、プロダクトテスト、プロダクト仕様文書の変更を含めてはいけない。

## 完了規約

- `.codex/skill-template.md`、`.codex/agent-template.toml`、対象 `SKILL.md` との矛盾がない時に完了とする。
- skill 本体に手順、標準 型、参照タイミング一覧、知識範囲一覧が残っていないことを確認する。
- ファイル名、agent 名、skill 名、command 以外の英語由来名詞が本文に残っていないことを確認する。
- 残した英語表記がある場合は、残存理由が ファイル名、agent 名、skill 名、command のいずれかで説明できることを確認する。
- 非対象規約と停止規約が混在していないことを確認する。
- ツール権限と出力義務が skill 側へ混入していないことを確認する。
- 削除対象に旧参照、空実体、禁止文言だけのファイルが残っていないことを確認する。
- 確認用ファイル、旧権限ファイル、agent 個別 JSON 規約の独立ファイルまたは参照が残っていない。

## 停止規約

- サブエージェントまたは別 agent から呼び出された場合は拒否する。
- プロダクトコードまたはプロダクトテストの変更が混ざる場合は停止する。
- プロダクト仕様文書の正本化が混ざる場合は停止する。
- agent TOML に 書き込み許可 / 実行許可 がない場合は停止し、不足項目名を返す。
- ツール権限、agent 所有者、責務境界が不明な場合は停止する。
- 残す英語表記が ファイル名、agent 名、skill 名、command のどれに当たるか判断できない場合は停止する。
- 削除理由または削除後の正本確認先がない削除は停止する。
