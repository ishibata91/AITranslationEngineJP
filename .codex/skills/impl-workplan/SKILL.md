# Impl Workplan

## Goal

- 実装順を固定する

## Rules

- 必要なら active exec-plan の `Implementation Plan` だけを更新してよい
- `tasks.md` を作らない
- section 数は最小限にする
- frontend / backend の責務境界を曖昧にしない
- 今の repo に不要な legacy schema を持ち込まない
