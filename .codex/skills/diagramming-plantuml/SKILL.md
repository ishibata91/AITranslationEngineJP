# Diagramming PlantUML

## ワークフロー

1. 図を独立した `.puml` ファイルにするか決める。
2. activity 図と component 図は分ける。
3. 既定では縦方向にする。横向きのほうが明確な時だけ変える。
4. `.puml` を書くか更新する。
5. `plantuml-check <file>` を実行する。
6. 構文エラーを先に直し、その後にレイアウトと読みやすさを整え、通るまで再実行する。

## ルール

- 関係のない図種を 1 ファイルに混ぜない。
- できるだけ線が交差しないようにする｡
- ラベルは短く、意味が明確なものにする。
- `plantuml-check` がエラーを返したら、その指摘を正として先に修正する。
- 大きいワークフローは 1 枚に詰めず、複数ファイルに分ける。

## 補足

- レンダリング画像が必要な場合でも、正本は `.puml` のままにする。画像出力は明示された時だけ行う。
