# Scenario Design Checklist

## Knowledge Check

- [ ] 必ず通す要件と risk を分けた
- [ ] user journey と scenario matrix を分けた
- [ ] 開始条件、操作、期待結果、観測点を明示した
- [ ] fake / fixture / validation command を確認した

## Common Pitfalls

- [ ] 実装方針を要件として固定しなかった
- [ ] paid な real AI API を前提にしなかった
- [ ] happy path だけにしなかった
- [ ] product test 実装詳細を書かなかった
