# Scenario テスト一覧: master-persona-management

- `対象`: マスターペルソナの一覧参照、検索、plugin 絞り込み、詳細表示、更新、削除、AI生成
- `topic_abbrev`: `MPM`

## ケース一覧

### SCN-MPM-001 一覧初期表示と同一画面詳細同期

- `分類`: 正常系
- `観点`: 独立ページで一覧、検索、詳細、AI生成パネルが同時に成立する
- `事前条件`: `master-persona` route からページへ到達できる
- `手順`:
  1. マスターペルソナページを開く。
  2. 一覧件数、選択中 entry、右側詳細の初期表示を確認する。
  3. 一覧の別 entry を選択し、詳細の同期を確認する。
- `期待結果`:
  1. AI設定、JSON preview、進行状況、一覧、詳細が同一ページに表示される。
  2. 一覧は生成状態一覧ではなく、ペルソナ要約を含むペルソナ一覧として表示される。
  3. 一覧選択の変更だけで詳細が即時に切り替わり、画面遷移を要求しない。
- `責務境界メモ`: 一覧選択は同一ページ状態更新で完結し、詳細は選択結果を表示するだけに留める。

### SCN-MPM-002 検索と plugin 絞り込みの再同期

- `分類`: 正常系
- `観点`: 検索と plugin 絞り込みを組み合わせても一覧と詳細が同期する
- `事前条件`: 複数 plugin、複数 race、複数 voice を含む entry が存在する
- `手順`:
  1. 検索欄にキーワードを入力する。
  2. 名前、FormID、EditorID、種族、voice のいずれかで一致する entry を確認する。
  3. 検索欄の右側 dropdown で plugin を切り替える。
- `期待結果`:
  1. 検索対象は `名前 / FormID / EditorID / 種族 / voice` に限定される。
  2. 右側 dropdown は plugin grouping / filter 専用であり、生成状態 filter としては使われない。
  3. 検索結果外または plugin 対象外になった選択中 entry は、表示対象内の entry へ切り替わるか空状態へ遷移する。
- `責務境界メモ`: 検索再計算と plugin 絞り込みは一覧責務とし、詳細は選択された 1 件だけを表示する。

### SCN-MPM-003 詳細のダイアログ数表示とダイアログ一覧モーダル

- `分類`: 正常系
- `観点`: 詳細からダイアログ数を確認し、モーダルを開閉できる
- `事前条件`: ダイアログを持つ entry を選択済み
- `手順`:
  1. 詳細初期表示時のダイアログ一覧モーダル状態を確認する。
  2. 詳細内の `ダイアログ一覧` を押す。
  3. モーダル内の行数表示とダイアログ本文を確認し、`閉じる` を押す。
- `期待結果`:
  1. モーダルは初期状態で閉じている。
  2. 詳細には `ダイアログ数` と `ダイアログ一覧` button が表示される。
  3. モーダルを開くと選択中 entry のダイアログ一覧を確認でき、閉じると元の詳細画面へ戻る。
- `責務境界メモ`: ダイアログ一覧は詳細から開く補助 UI とし、一覧本体に常時展開しない。

### SCN-MPM-004 欠落属性を persona-facing descriptor として露出しない

- `分類`: 正常系
- `観点`: race / sex が欠落していても、欠落値をペルソナ向け表示へ混ぜない
- `事前条件`: `race` または `sex` が未設定で、利用可能な name attribute も不足した entry が存在する
- `手順`:
  1. 対象 entry を一覧から選択する。
  2. 一覧のラベルと詳細のタグ表示を確認する。
  3. voice やダイアログ数など利用可能な属性が残る表示を確認する。
- `期待結果`:
  1. 欠落した `race / sex` を `不明` などの persona-facing descriptor として表示しない。
  2. 利用可能な属性だけで一覧と詳細の表示を構成する。
  3. 生成継続用の内部基底があっても、欠落属性をそのまま人物ラベルとして露出しない。
- `責務境界メモ`: raw 属性の欠落処理は presenter / screen state で吸収し、表示テキストへ未設定値を流し込まない。

### SCN-MPM-005 更新モーダル

- `分類`: 正常系
- `観点`: 詳細の `更新` から選択中 entry を編集できる
- `事前条件`: run が active ではなく、更新対象 entry を選択済み
- `手順`:
  1. 詳細の `更新` を押す。
  2. ペルソナ本文または識別属性を変更する。
  3. `更新する` を押す。
- `期待結果`:
  1. 選択中 entry の値が事前入力された editor が開く。
  2. 保存後は editor が閉じる。
  3. 一覧の要約と詳細本文の両方が更新後の値に同期し、選択状態を維持する。
- `責務境界メモ`: update 起点は詳細側だが、結果反映は一覧と詳細の同一 state で完結する。

### SCN-MPM-006 削除確認モーダル

- `分類`: 正常系
- `観点`: 詳細の `削除` から確認付きで削除できる
- `事前条件`: run が active ではなく、削除対象 entry を選択済み
- `手順`:
  1. 詳細の `削除` を押す。
  2. 確認モーダルで対象を確認する。
  3. `削除する` を押す。
- `期待結果`:
  1. 削除対象の名前、FormID、EditorID を確認できる。
  2. 確定後は対象 entry が一覧から除外される。
  3. 詳細は次の visible entry へ切り替わるか、残存なしなら空状態へ切り替わる。
- `責務境界メモ`: 削除確定は確認モーダル責務とし、一覧の直接即時削除は行わない。

### SCN-MPM-007 JSON 選択と AI生成 preview

- `分類`: 正常系
- `観点`: `extractData.pas` JSON を選択し、skip 前提の preview を確認できる
- `事前条件`: 型に合う JSON を選択できる
- `手順`:
  1. `JSON を選ぶ` を押してファイルを選ぶ。
  2. 対象ファイル名、対象 plugin、総 NPC 数、今回生成する数、既に作成済み、会話が見つからない、汎用NPC の表示を確認する。
  3. 開始前メッセージと button 文言を確認する。
- `期待結果`:
  1. 未選択時は開始操作が表示されないか無効化される。
  2. 選択後に preview card が更新され、create / rebuild / overwrite の文言は出ない。
  3. `作成済みのペルソナはスキップされます` の文言が表示される。
- `責務境界メモ`: preview の対象件数と skip 内訳は、ファイル選択状態と事前照合結果に閉じる。

### SCN-MPM-008 生成中の進行状況と update / delete lock

- `分類`: 状態遷移
- `観点`: 長時間処理中でも進捗と lock 状態を観測できる
- `事前条件`: AI生成を開始済み
- `手順`:
  1. 生成開始後の run panel を確認する。
  2. 進捗バー、完了件数、現在処理中 NPC、現在の状態を確認する。
  3. 詳細の action 文言と button 状態を確認する。
- `期待結果`:
  1. run panel に progress、完了件数、現在処理中 NPC、状態が表示される。
  2. 生成中は `更新と削除を行えません` と表示され、`更新` と `削除` は操作できない。
  3. 一覧と詳細は読み続けられ、画面遷移なしで状態観測を継続できる。
- `責務境界メモ`: 長時間処理の可視化は run panel 責務とし、lock は detail action に反映する。

### SCN-MPM-009 生成完了後の same-page 再同期

- `分類`: 状態遷移
- `観点`: 完了後に一覧、選択、詳細、action 可否が一貫して更新される
- `事前条件`: AI生成が完了している
- `手順`:
  1. 完了メッセージと完了件数、skip 件数を確認する。
  2. 一覧と詳細が対象 plugin の最新状態へ更新されることを確認する。
  3. 詳細上部の action 可否文言を確認する。
- `期待結果`:
  1. 完了後も overwrite 結果ではなく、作成件数と skip 内訳が表示される。
  2. 一覧、選択、詳細が同一ページ内で再同期し、画面再読み込みを要求しない。
  3. 生成完了後は `更新と削除を行えます` と表示され、button が再び有効になる。
- `責務境界メモ`: 完了時の再同期は screen state が一括適用し、route 遷移や全画面再初期化へ逃がさない。

### SCN-MPM-010 JSON 不正時の失敗表示

- `分類`: 主要例外系
- `観点`: `extractData.pas` JSON 不正時の失敗を同一ページで扱う
- `事前条件`: 不正な JSON を選択済み
- `手順`:
  1. 不正 JSON を選択して生成前確認または生成開始を行う。
  2. 失敗メッセージと再実行可否を確認する。
  3. ファイルを維持したまま再試行できるかを確認する。
- `期待結果`:
  1. 状態が `入力エラー` または `失敗` へ遷移する。
  2. 原因例と戻り先を同じカードで確認できる。
  3. 失敗後もページ遷移せず、再試行または選び直しへ進める。
- `責務境界メモ`: 失敗時は error page へ遷移せず、AI生成カード内の状態として閉じる。
