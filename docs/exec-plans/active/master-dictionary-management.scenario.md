# Scenario テスト一覧: master-dictionary-management

- `task_id`: `master-dictionary-management`
- `usecase`: `master-dictionary-management`
- `final_artifact_path`: `docs/scenario-tests/master-dictionary-management.md`
- `対象`: マスター辞書の一覧参照、検索、詳細表示、作成、更新、削除、XML取り込み

## ケース一覧

### SCN-MDM-001 一覧初期表示と同一画面詳細表示

- `分類`: 正常系
- `観点`: 独立ページで一覧 30 件表示と詳細同期が成立する
- `事前条件`: マスター辞書ページへ到達できる
- `手順`:
  1. マスター辞書ページを開く。
  2. 一覧の 1 ページ表示件数と選択状態を確認する。
  3. 一覧の別行を選択し、右側詳細の表示内容を確認する。
- `期待結果`:
  1. 一覧は 1 ページ 30 件で表示される。
  2. 選択中エントリの原文と訳語が同一画面の詳細へ即時反映される。
  3. 画面遷移せず、一覧と詳細の状態を同時に確認できる。
- `責務境界メモ`: 一覧操作は同一ページ内状態更新として扱い、別画面遷移を要求しない。

### SCN-MDM-002 検索とカテゴリ絞り込み

- `分類`: 正常系
- `観点`: 検索語とカテゴリ条件で対象を絞り込める
- `事前条件`: 一覧に複数カテゴリの辞書エントリが存在する
- `手順`:
  1. 検索欄へ原文または訳語の一部を入力する。
  2. カテゴリを選択して絞り込む。
  3. 条件を変更し、表示件数と選択状態の更新を確認する。
- `期待結果`:
  1. 検索条件とカテゴリ条件の両方を満たす一覧へ更新される。
  2. 表示件数の文言が絞り込み結果へ追従する。
  3. 選択中エントリが条件外になった場合は表示対象内のエントリへ切り替わる。
- `責務境界メモ`: 絞り込み結果は一覧領域の責務で確定し、詳細表示は選択状態の結果だけを表示する。

### SCN-MDM-003 新規登録モーダル

- `分類`: 正常系
- `観点`: 一覧ヘッダーから新規登録モーダルを開き保存できる
- `事前条件`: マスター辞書ページが表示されている
- `手順`:
  1. `新規登録` を押して編集モーダルを開く。
  2. 原文、訳語、カテゴリ、由来を入力する。
  3. `保存する` を押す。
- `期待結果`:
  1. 新規登録用モーダルが表示される。
  2. 保存後にモーダルが閉じる。
  3. 同一ページ内で新規エントリが一覧へ追加され、詳細表示は新規エントリへ同期するか未選択状態ルールに従って更新される。
- `責務境界メモ`: 作成導線は一覧ヘッダー起点、入力は編集モーダル責務で完結する。

### SCN-MDM-004 更新モーダル

- `分類`: 正常系
- `観点`: 詳細の `更新` から編集モーダルで内容変更できる
- `事前条件`: 一覧で更新対象エントリを選択済み
- `手順`:
  1. 詳細の `更新` を押す。
  2. モーダルで訳語またはカテゴリを変更する。
  3. `保存する` を押す。
- `期待結果`:
  1. 選択中エントリの値が事前入力された編集モーダルが開く。
  2. 保存後にモーダルが閉じる。
  3. 同一ページ内で一覧行と詳細表示の両方が更新後の値へ同期し、再選択や再遷移を要しない。
- `責務境界メモ`: 更新起点は詳細領域、更新結果の反映先は一覧と詳細の同一状態。

### SCN-MDM-005 削除確認モーダル

- `分類`: 正常系
- `観点`: 詳細の `削除` から確認モーダル経由で削除できる
- `事前条件`: 一覧で削除対象エントリを選択済み
- `手順`:
  1. 詳細の `削除` を押す。
  2. 削除確認モーダルで対象情報を確認する。
  3. `削除する` を押す。
- `期待結果`:
  1. 削除確認モーダルが表示され、対象エントリを確認できる。
  2. 確定後にモーダルが閉じ、同一ページ内で対象エントリが一覧から除外される。
  3. 詳細は残存エントリへ切り替わるか、残存なしの場合は空状態表示へ切り替わる。
- `責務境界メモ`: 削除確定は確認モーダル責務とし、一覧の直接即時削除は行わない。

### SCN-MDM-006 XML取り込み導線

- `分類`: 正常系
- `観点`: ファイル選択から取込バー表示、取り込み開始、完了表示まで同一画面で確認でき、許可リスト対象 REC だけを単語抽出する
- `事前条件`: マスター辞書ページが表示され、`/Users/iorishibata/Repositories/AITranslationEngineJP/dictionaries/Dawnguard_english_japanese.xml` を選択できる
- `手順`:
  1. `ファイルを選択` で XML ファイルを選ぶ。
  2. 取込バーに対象ファイル名と状態表示が出ることを確認する。
  3. `この XML を取り込む` を押し、進捗表示の変化と完了表示を確認する。
  4. 取り込み結果に含まれる REC 種別を確認する。
- `期待結果`:
  1. ファイル選択後にのみ取込バーが表示され、未選択状態では取込開始操作を表示しない。
  2. 取込バーは `取込待ち` から `取込中` を経て `完了` へ遷移し、進捗表示が同一画面で更新される。
  3. 完了後は検索語とカテゴリ絞り込みが解除され、一覧は先頭 30 件へ戻り、取り込み対象エントリが再選択されて詳細へ表示される。
  4. 単語抽出対象 REC は `BOOK:FULL`, `NPC_:FULL`, `NPC_:SHRT`, `ARMO:FULL`, `WEAP:FULL`, `LCTN:FULL`, `CELL:FULL`, `CONT:FULL`, `MISC:FULL`, `ALCH:FULL`, `FURN:FULL`, `DOOR:FULL`, `RACE:FULL`, `INGR:FULL`, `FLOR:FULL`, `SHOU:FULL` のみであり、`ACTI:FULL` や `MGEF:DNAM` など許可リスト外 REC は抽出しない。
- `責務境界メモ`: XML取り込みは専用カード内で状態を閉じ、別画面へ遷移しない。抽出対象の判定は `REC` 値の許可リストで行う。

### SCN-MDM-007 検索結果ゼロ件

- `分類`: 主要例外系
- `観点`: 一致なし検索時の空状態表示
- `事前条件`: マスター辞書ページが表示されている
- `手順`:
  1. 一致しない検索語を入力する。
  2. 一覧表示とメッセージを確認する。
  3. 検索語を解除する。
- `期待結果`:
  1. 一覧に空状態メッセージが表示される。
  2. 異常終了せず、同一画面で条件変更を継続できる。
  3. 条件解除で通常一覧へ戻る。
- `責務境界メモ`: 例外系はエラー画面遷移ではなく、一覧領域の空状態として扱う。

### SCN-MDM-008 XML未選択時の取込開始操作ゲート

- `分類`: 主要例外系
- `観点`: ファイル未選択時は取込開始操作へ進めない
- `事前条件`: XML 未選択
- `手順`:
  1. マスター辞書ページを開き、ファイル未選択状態を確認する。
  2. 取込バーと `この XML を取り込む` 操作の表示有無を確認する。
  3. `ファイルを選択` で XML を選択し、取込開始操作が表示されることを確認する。
- `期待結果`:
  1. 未選択時は取込バーが表示されない。
  2. 未選択時は取込開始操作が表示されない。
  3. ファイル選択後にのみ取込開始操作が表示される。
- `責務境界メモ`: 取り込み開始可否の前提判定はファイル選択状態で決まり、未選択時に開始イベントを発火しない。

### SCN-MDM-009 取込バー状態遷移

- `分類`: 状態遷移
- `観点`: 取込バーの状態が `待機中` -> `取込待ち` -> `取込中` -> `完了` へ遷移し、完了時に同一ページ再同期ポリシーが適用される
- `事前条件`: マスター辞書ページが表示されている
- `手順`:
  1. XML を選択して `取込待ち` 状態を確認する。
  2. 取り込み開始後に `取込中` と進捗変化を確認する。
  3. 完了表示を確認し、一覧・選択・詳細の更新内容を確認する。
  4. `選び直す` で初期状態へ戻す。
- `期待結果`:
  1. 状態表示は `待機中` -> `取込待ち` -> `取込中` -> `完了` の順序で遷移する。
  2. 完了後は検索語とカテゴリ絞り込みが解除され、画面再遷移を要せず同一ページで再同期される。
  3. 完了後の一覧は先頭 30 件へ戻り、一覧件数と一覧内容が取り込み結果へ更新される。
  4. 完了後は取り込み対象エントリが再選択され、詳細表示で取り込み結果を確認できる。
  5. 選び直し操作で `待機中` と未選択表示へ戻る。
- `責務境界メモ`: 取込バー状態は独立管理し、完了時は検索/絞り込み解除、先頭 30 件再着座、取込対象再選択を一貫適用する。

### SCN-MDM-010 編集導線の状態遷移

- `分類`: 状態遷移
- `観点`: 新規登録モーダルと更新モーダルの開閉状態
- `事前条件`: 一覧で任意エントリを選択済み
- `手順`:
  1. `新規登録` でモーダルを開き、`閉じる` を押す。
  2. 詳細の `更新` でモーダルを開き、`閉じる` を押す。
- `期待結果`:
  1. どちらの導線でも編集モーダルを開閉できる。
  2. モーダルを閉じても一覧選択と詳細表示は維持される。
- `責務境界メモ`: モーダル開閉は入力導線責務であり、一覧選択状態を不用意に変更しない。

### SCN-MDM-011 画面責務境界の維持

- `分類`: 責務境界
- `観点`: 一覧、詳細、モーダル、取込バーの責務が同一ページで分離される
- `事前条件`: マスター辞書ページが表示されている
- `手順`:
  1. 一覧選択、更新モーダル、削除モーダル、XML取り込み導線を順に操作する。
  2. 各操作中に他領域の表示崩れや不意な画面遷移がないか確認する。
- `期待結果`:
  1. 各導線はそれぞれの領域内で状態を持ち、操作対象を跨いで破綻しない。
  2. 独立ページのまま、同一画面で操作と状態確認を完結できる。
- `責務境界メモ`: 本 task はマスター辞書単独ページ内完結を証明対象とし、ダッシュボード配下連携を含めない。
